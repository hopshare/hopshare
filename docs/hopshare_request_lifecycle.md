# **Hopshare Request Lifecycle (Draft)**

## Request Structure and Lifecycle

Requests are the lifeblood of Hopshare. Without a clearly defined and simple process, Hopshare will not work.

Members create Requests in order to ask for help from other Members. Because people typically feel uncomfortable asking others for help, the whole idea of a time bank is to create some structure around the process to make it feel less intimidating. We use the idea of 'credits' in each Member's time bank account to moderate how Requests are made. Each Request must provide the following information:

* An estimated number of hours needed.
* A description of what is being requested as free form text.
* The timeframe for when the request needs to be fulfilled (anytime, before/around or on a specific date and time).
* Optional details beyond the description that may be relevant.

Once created, a Request is publicly available within the Organization for any Member to view. Hopshare will attempt to match Members who might be qualified or interested in helping (see below). At this point, the Request can potentially be in one of six states:

* Created: the requesting Member has submitted the Request to the Organization.
    * Accepted: another Member has indicated they would like to fulfil the request.
      * Expired: the timeframe for the request has passed.
      * Completed: the request has been fulfilled. This is a terminal state.
    * Interested: another Member has indicated they are interested in helping but want additional information.
      * Canceled
      * Expired
* Canceled: the requesting Member has withdrawn the Request. This is a terminal state.
* Expired: the timeframe for the request has passed and it is no longer relevant. This is a terminal state.


## Fulfilling Requests and Member Communication

After a Member accepts Request, the two Members need to coordinate on the logistics to get the Request fulfilled. Privacy is a major concern on Hopshare, and it is important that we do not require Members to share any more information than necessary. We will suggest to Members that they do not share public information in their requests (such as telephone numbers, addresses, or emails). Members will specify in their profile what their preferred method of contact is (phone number to call or text, or email). When a Request has been Accepted, Hopshare will provide the accepting Member the requesting Member's preferred contact information. From this point on, it is up to the two Members to coordinate on the Request.

Once the Request has been fulfilled, it is important for one or both Members to record the activity in Hopshare. This ensures there is a record of the Request being completed so that credits can be properly exchanged. Either Member marks the Request as Complete and adds a comment (required) and optionally pictures. Completed Requests are publicly available in the Organization so they can be celebrated by other Members with comments. A success goal for each Hopshare Organization is to have a vibrant, full list of Completed Requests that can be viewed to inspire others to help each other.

If a Request is fulfilled but neither Member updates the status, Hopshare will email both users asking them to update it. If this still has not occurred after a reasonable amount of time (e.g. a week), Hopshare will mark the Request as Complete with a standard comment but no credits will be transferred. This motivates Members to update their Requests in a timely fashion.
